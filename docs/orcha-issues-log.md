## How to add an entry
New issue тЖТ add a row (next `ISS-n`), set status, one-line detailтЖТfix. When fixed, flip to ЁЯЯй and note the PR/commit. Pair with `docs/orcha-functional-tests.md` (a regression test per fixed issue where it makes sense).
**Subtask convention:** a gap that is a child of an existing issue is numbered `ISS-N.M` (e.g. ISS-8.1, ISS-8.2 under ISS-8) and tagged *(subtask of ISS-N)* тАФ the log is a flat markdown table, so hierarchy is by this naming convention. (Orcha's task model itself has a dependency DAG + `is_root` but no real parent/child containment тАФ true in-product subtasks would be a feature тАФ **PV1-1** in `docs/orcha-postv1-plan-and-tasks.md`.)
